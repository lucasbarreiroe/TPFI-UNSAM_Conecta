## Directory Structure Rationale

### Backend Organization
- **Layered Architecture**: Routes → Controllers → Services → Repositories
  - Enables testability and separation of concerns
  - Services contain business logic (e.g., "prevent overbooking via atomic updates")
  - Repositories abstract database queries

- **Jobs Folder**: `notificationScheduler.ts` will run cron tasks to:
  - Check events starting in 24 hours → send reminders
  - Check events starting in 1 hour → send urgent reminders
  - Based on `preferredNotificationChannel` (EMAIL, PHONE)

- **Validators**: Input validation using libraries like `joi` or `zod` for schema validation

- **Tests**: BDD-style tests using Jest with "GIVEN-WHEN-THEN" structure

### Frontend Organization
- **Pages**: Route-level components (CalendarPage, DashboardPage, etc.)
- **Components**: Reusable UI components organized by feature domain
- **Hooks**: Custom React hooks for data fetching and state management
- **Services**: API client methods for communicating with backend
- **Contexts**: Global state management (Auth, Notifications)
- **Types**: Centralized TypeScript interfaces

### Deployment
- **Docker**: Separate images for backend and frontend for scalability
- **Render.yaml**: Infrastructure as code for deployment
- **Scripts**: Automated migration and seeding
